# Indie Pigeon – Release v1.0.0

**Release Date:** 2026-03-10
**Status:** Production Deployment
**Codename:** Digital Delivery MVP

---

# Overview

Version **v1.0.0** marks the first complete production deployment of the Indie Pigeon digital delivery system.

The platform successfully automates the full workflow:

1. Email capture
2. Payment processing
3. Webhook verification
4. Secure file delivery
5. Automated email fulfillment

This release establishes the baseline architecture for a creator-focused digital product delivery platform.

---

# Core Features

## Payment Integration

* Square Checkout integration
* Webhook verification using HMAC signatures
* Payment status validation
* Duplicate fulfillment protection

## Order Management

* Persistent order storage
* PostgreSQL database via Neon
* Order lifecycle tracking

Order states:

```
pending
paid
fulfilled
failed
```

---

## Secure Download System

Digital files are delivered using signed URLs.

Security features include:

* HMAC-SHA256 signed download URLs
* Expiring links
* Disabled public file routes
* Secure Cloudflare Worker validation
* File delivery from Cloudflare R2 storage

Download URL format:

```
/dl?key=<file>&exp=<timestamp>&sig=<signature>
```

---

## Email Delivery

Emails are sent using the **Resend API**.

Features:

* transactional email delivery
* verified sending domain
* download link distribution
* provider message ID logging

---

## Manual Recovery Tool

A protected manual email resend endpoint was implemented.

Endpoint:

```
/manual-send
```

Features:

* password protected
* regenerate secure download link
* resend ebook to customer

This ensures customer fulfillment even if webhook delivery fails.

# Database Schema

Tables created automatically at startup:

```

```

Orders track:

* checkout reference
* payment ID
* buyer email
* fulfillment state
* timestamps

Delivery logs track:

* email status
* provider message ID
* errors

# Known Limitations

Current MVP limitations:

* orders matched using latest pending email capture
* no admin dashboard
* no purchase history interface
* no download analytics
* single product configuration

These will be addressed in future releases.
